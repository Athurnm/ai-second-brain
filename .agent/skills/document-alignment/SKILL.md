# Document Alignment Skill

## Purpose
When working with multiple related documents (e.g., financial model + GTM strategy), ensure all numbers, targets, and timelines are consistent and coherent.

## When to Use
- After creating or updating any financial model, GTM, or roadmap document
- When user requests changes to targets, budgets, or timelines
- When user asks to "align" or "sync" documents

## Process

### Step 1: Read All Related Documents
Before making any changes, read ALL related documents in full to understand:
- Key metrics and targets
- Timelines and milestones
- Budget allocations
- Assumptions

### Step 2: Create Alignment Matrix
Document the key metrics that must match across documents:

| Metric | Financial Model | GTM | Roadmap |
|--------|-----------------|-----|---------|
| New Users (period) | | | |
| Budget (period) | | | |
| GMV Targets | | | |
| CPI | | | |
| Feature Launch Dates | | | |

### Step 3: Identify Misalignments
Flag any inconsistencies:
- Different totals for same metric
- Conflicting timelines
- Budget allocations that don't add up
- Feature dependencies not reflected

### Step 4: Determine Source of Truth
Establish which document is the source of truth for each metric:
- **Financial Model**: Budget, users, GMV, margins, unit economics
- **GTM Strategy**: Channel mix, campaigns, messaging, tactics
- **Roadmap**: Feature timelines, dependencies

### Step 5: Make Changes
Update documents to align, always deferring to the source of truth.

### Step 6: Verify Alignment
Re-read both documents and confirm all key metrics match.

## Key Alignment Points

### Financial Model → GTM
- Total users per period MUST match
- Total budget per period MUST match
- CPI targets MUST match
- GMV/MTU targets MUST match
- Retention rates should match

### GTM → Roadmap
- Feature launches in GTM MUST align with roadmap quarters
- Campaign messaging should reflect available features
- P2P, gamification, virtual card timing must match

### Common Mistakes to Avoid
1. Updating one document without the other
2. Using different period definitions (quarter vs month)
3. Forgetting to update summary tables
4. Mismatched retention/CVR assumptions
5. Budget categories that don't add up to totals

## Verification Checklist
Before completing, verify:
- [ ] User totals match across documents
- [ ] Budget totals match across documents
- [ ] Period-by-period breakdown is consistent
- [ ] Summary sections reflect detail sections
- [ ] Success metrics match between documents
- [ ] Feature timing aligns with roadmap
